# Лабораторная работа №3

## Задания:

1. Написать “плохой” CI/CD файл, который работает, но в нем есть не менее пяти “bad practices” по написанию CI/CD
2. Написать “хороший” CI/CD, в котором эти плохие практики исправлены
3. В Readme описать каждую из плохих практик в плохом файле, почему она плохая и как в хорошем она была исправлена, как исправление повлияло на результат

## CI/CD

* Перед выполнение лабораторной работы вкратце разберемся, что из себя представляет аббревиатура "CI/CD". CI/CD (Continuous Integration, Continuous Delivery — непрерывная интеграция и доставка) - это технология автоматизации тестирования и доставки кода.
